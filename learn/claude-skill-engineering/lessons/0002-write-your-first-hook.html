Claude Skill Engineering · Lesson 02

写第一个 Hook

从零写一个真实 hook，装进你的 settings.json，跑起来看效果

Stop hook settings.json osascript

这节课做什么

写一个 Stop hook：每次 Claude 完成响应后，Mac 桌面弹出一条通知。

为什么选这个作为第一个 hook：

  • Stop 事件最简单——不需要 matcher，不需要拦截工具，就是"完成了就触发"
  • 效果立即可见，不用猜有没有生效
  • 实际有用：跑长任务时你可以离开，完成后收到通知

1. 先看你现有的 settings.json

你的关税查询 hook 已经在里面了。先看一眼现有结构，再往里加新 hook。

1

打开文件：

open ~/.claude/settings.json
2

找到 "hooks" 字段，看清楚现有结构。大概长这样：

{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "查关税",
        "hooks": [{ "type": "command", "command": "..." }]
      }
    ]
  }
}

2. 加入 Stop Hook

在现有的 hooks 对象里，加一个 "Stop" 字段。完整结构如下：

{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "查关税",
        "hooks": [{ "type": "command", "command": "..." }]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude 完成了\" with title \"Claude Code\" sound name \"Glass\"'"
          }
        ]
      }
    ]
  }
}
注意：Stop hook 不需要 matcher 字段——每次 Claude 停止响应都触发，没有条件。

3. 验证是否生效

1

保存 settings.json

2

在 Claude Code 里发任意一条消息，等它回完

3

右上角应该弹出通知 "Claude 完成了"，带一声提示音

如果没弹：检查系统设置 → 通知 → 终端/Claude Code 是否允许通知

4. 解剖这个 Hook

{
  "type": "command",          // 目前唯一的 hook 类型
  "command": "osascript ..."  // 任意 shell 命令，这里用 macOS 的通知 API
}

osascript 是 macOS 执行 AppleScript 的命令行工具。display notification 是弹通知的 AppleScript 语句。换成其他命令也完全可以——发 Slack、写日志、执行脚本都行。

关键认知：hook 的 command 就是一条 shell 命令，和你在终端里手敲没有区别。Claude 不参与这个过程——是 harness（Claude Code 运行时）直接执行的。

5. 三个实用变体（选一个试试）

变体 A：通知里带时间戳

"command": "osascript -e \"display notification \\\"完成于 $(date '+%H:%M:%S')\\\" with title \\\"Claude Code\\\"\""

变体 B：写日志到文件

"command": "echo \"$(date '+%Y-%m-%d %H:%M:%S') Claude 完成\" >> ~/claude-sessions.log"

日志会追加到 ~/claude-sessions.log，可以用 tail -f ~/claude-sessions.log 实时查看。

变体 C：两个一起（通知 + 日志）

"command": "osascript -e 'display notification \"Claude 完成了\" with title \"Claude Code\" sound name \"Glass\"' && echo \"$(date '+%Y-%m-%d %H:%M:%S') done\" >> ~/claude-sessions.log"

理解检验

Q1. Stop hook 里为什么不需要写 matcher？

Q2. 如果你想让 hook 只在用户消息含有"广告优化"时才执行某个脚本，应该用哪个事件 + 什么 matcher？

Q3. hook 的 command 执行失败（脚本报错）会怎样？

下一节

你现在会写 Stop hook 了。下一步是 PreToolUse hook——在 Claude 调用某个工具前拦截，做审计或注入信息。这是 hook 里最有工程价值的一种，也是最容易踩坑的。
