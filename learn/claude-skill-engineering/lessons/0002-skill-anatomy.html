Claude Skill Engineering · Lesson 02

Skill 解剖

一个好的 skill 由哪些部分组成，各部分负责什么，缺了会怎样

SKILL.md 结构 诊断

1. 一个 skill 为什么会"不好用"

你一定遇到过这些情况：

  • 明明说了要用某个 skill，Claude 没触发
  • 同一个 skill，今天输出和昨天格式不一样
  • 输出内容对，但啰嗦、需要大量手改
  • skill 越写越长，但效果反而变差

这些问题背后，几乎都能追溯到 skill 某个部分写得不对或缺失。

2. 一个 skill 的四个部分

SKILL.md
amazon-listing-v2（示例）
① 触发描述
必须有
frontmatter 里的 description 字段。Claude 靠它决定"这条消息该不该用这个 skill"。
写错 → 不触发 / 误触发 / 和别的 skill 打架
② IPO 契约
必须有
明确写：输入什么 → 处理什么 → 输出什么。
就像告诉一个新员工：你收到什么材料、要做什么、要交什么东西。
缺失 → 输出格式飘，每次不一样
③ 执行指令
必须有
具体怎么做的步骤、规则、判断标准。这是 skill 的主体内容。
写得模糊 → 输出质量不稳定，靠 Claude 自由发挥
④ 范例
强烈建议
1-2 个完整的"输入 → 输出"真实样本。Claude 会照着结构输出，比写一百条规则更有效。
没有 → 格式靠猜，每次不同

3. 重点讲：触发描述（最容易出问题）

description 是 Claude 判断要不要启用这个 skill 的唯一依据。有两个常见错误：

错误写法：把描述写成介绍，触发词埋在后面

"这是一个专业的亚马逊 Listing 撰写工具，基于 COSMO 语义和 A9 算法，帮助卖家优化标题、五点描述……当用户需要写 listing 时触发"

问题：Claude 读 description 时有字数限制，后半段可能被截掉，触发词没了。
正确写法：第一句就是触发场景

"当用户需要撰写或优化亚马逊 Listing 时触发，包括写标题、五点、描述……"

触发词前置，不管截不截都能命中。

4. 重点讲：IPO 契约（决定输出稳不稳定）

IPO = Input（输入）/ Process（处理）/ Output（输出）。

一个没有 IPO 契约的 skill，就像你雇了个助理，但没告诉他"你负责什么、用什么格式交给我"——每次结果都靠他自己发挥。

部分 写什么 举例
Input 用户需要提供什么 产品名称、目标市场、核心卖点、竞品 ASIN
Process skill 要做哪些步骤 分析竞品关键词 → 提炼差异化卖点 → 套用 A9 规则写标题
Output 交付物是什么，格式是什么 Title（200字符内）+ 5 条 Bullet Points + Search Terms
IPO 不用长，几行就够。它的价值是：你和 Claude 都清楚"这个 skill 做到什么为止"。

5. 用这个框架诊断 skill 问题

症状 大概率出在哪 怎么告诉 Claude 修
说了要用，但没触发 ① 触发描述：触发词不够明确，或埋太深 "帮我把这个 skill 的 description 第一句改成触发场景"
两个 skill 抢着触发 ① 触发描述：两个 skill 的触发词重叠 "帮我检查这两个 skill 的 description 有没有冲突，把次要的那个限定得更窄"
每次格式不一样 ② IPO 契约 + ④ 范例：输出没有约束 "帮我加一个输出模板，并加一个完整的范例"
输出质量飘，靠运气 ③ 执行指令：规则太模糊，没有判断标准 "把这条指令从模糊描述改成具体的 if-then 规则"
skill 越长越差 ③ 执行指令：把所有细节都塞进主文件 "帮我把细节内容拆到 references 文件夹，主文件只保留骨架"

诊断练习

Q1. 你的 amazon-ad-optimizer skill，用了几次发现每次输出的报告格式都不太一样——有时有表格，有时没有。这个问题出在哪个部分？

Q2. 你有两个 skill：amazon-listing-v2 和 listing-rufus-cosmo-audit。用户说"帮我优化这个 listing"，结果触发了错误的那个。问题在哪？

Q3. 以下哪种做法最能稳定 skill 的输出格式？

课后实操（可选）

打开你任意一个现有 skill 的 SKILL.md，对照四个部分检查：

  • ① 触发描述的第一句是不是触发场景？
  • ② 有没有写清楚输入什么、输出什么？
  • ③ 执行指令是具体规则还是模糊描述？
  • ④ 有没有一个完整的范例？

发现问题就告诉我，我帮你改。
