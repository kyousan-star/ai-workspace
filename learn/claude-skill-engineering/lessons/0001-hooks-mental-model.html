Claude Skill Engineering · Lesson 01

Hooks 是什么，什么时候用

理解 hooks 的心智模型——让行为自动触发，而不是靠指令提醒

hooks settings.json 事件驱动

1. 从一个真实问题出发

你在 CLAUDE.md 里写了"消息含'查关税'→ 自动启动关税查询工具"。这个行为是怎么触发的？

不是 Claude 读懂了你的意图后"主动"去做——而是 settings.json 里的 hook 在监听每一条用户输入，匹配到关键词后，在 Claude 响应之前就执行了一段 shell 命令。

关键区别：
SKILL.md / CLAUDE.md 里的指令 → 影响 Claude 怎么回答
Hooks → 影响 Claude 回答之前/之后/工具调用时，系统层面发生什么

2. Hook 的本质

Hook 是写在 settings.json 里的 shell 命令，绑定到 Claude Code 的生命周期事件上。每次事件触发，harness（运行时）自动执行对应命令——Claude 不参与这个决策。

配置位置：

// ~/.claude/settings.json  （全局）
// .claude/settings.json    （项目级，优先级更高）

{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "查关税",
        "hooks": [
          {
            "type": "command",
            "command": "cd ~/tariff-tool && node index.js"
          }
        ]
      }
    ]
  }
}

3. 四种事件类型

事件 触发时机 典型用途
UserPromptSubmit 用户发送消息后，Claude 响应前 关键词路由、预处理输入、注入上下文
PreToolUse Claude 调用某个工具前 拦截危险操作、记录审计日志、注入参数
PostToolUse 工具执行完毕后 结果后处理、触发下游任务、通知
Stop Claude 完成一轮响应后 输出摘要、自动提交、清理临时状态

4. matcher：什么情况下触发

matcher 是一个字符串（支持正则），对应不同事件有不同的匹配目标：

事件 matcher 匹配的内容
UserPromptSubmit 用户输入的消息文本
PreToolUse / PostToolUse 工具名称（如 Bash、Edit、Write）
Stop 无 matcher，每次停止都触发
不写 matcher = 匹配所有。例如 Stop 通常不需要 matcher。

5. 什么时候用 Hook，什么时候用 SKILL.md 指令

用 Hook 用 SKILL.md 指令
需要无论如何都执行（Claude 忘不了） 需要 Claude 理解上下文后决定是否执行
需要执行 shell 命令 需要 Claude 生成内容、做判断
响应时机要精确（工具调用前/后） 行为描述即可，不关心精确时机
"每次 Claude 停止时发通知" "回复时用简体中文"
陷阱：把 "每次 X 就 Y" 写进 CLAUDE.md 而不是 hooks，Claude 有时会忘。如果行为必须可靠，用 hook。

Q2. 你想让 Claude 在分析广告数据时，用专业的 PPC 术语而非口语化表达。用什么实现？
