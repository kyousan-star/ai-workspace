# Decision Log Schema

`01_WORKFLOW/decisions.md` 用于记录用户确认和关键取舍。

建议格式：

```text
# Decisions

| 时间 | 阶段 | 决策 | 理由 | 影响文件 |
|---|---|---|---|---|
```

## 必须记录

- Gate 1 是否通过。
- Gate 1 用户修改意见。
- **Gate 1 用户锁定的卖点优先级顺序**（后续所有阶段的图序唯一驱动来源）。
- **Gate 1 用户否决的视觉手段条目**（写入阶段4 STRICTLY AVOID）。
- **Gate 1 用户声明的视觉禁区**（写入阶段4 STRICTLY AVOID）。
- Gate 2 是否通过。
- Gate 2 图片序列调整。
- Gate 3 最终候选选择。
- 产品事实冲突的处理方式。
- 合规风险的处理方式。
- 放弃某个图片方向的原因。
