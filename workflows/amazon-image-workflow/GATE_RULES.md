# Gate Rules

Gate 是必须停下来让用户确认的节点。

## Gate 1：图片策略确认

触发时机：并行研究完成，图片策略已形成。

主控 Agent 在 Gate 1 展示以下内容，用户逐项确认：

**A. 产品事实核对**
- 产品外观、结构、核心配件、功能描述是否正确？
- 有没有 agent 标注的不确定项需要用户澄清？

**B. 卖点优先级确认（最关键）**
- 呈现候选卖点清单（来自 VOC/卖点 Agent），标注每个卖点的差异化程度、VOC 提及频率、视觉可感知性。
- 用户定义优先级顺序：哪个最先讲、哪些需要独立一张图、哪些可合并或靠后、哪些不在图里讲。
- **此处确认的卖点优先级即为后续图序的唯一驱动来源，图序规划 Agent 和 Brief/Prompt Agent 不得自行更改。**

**C. 品类基准线核对（来自竞品 Part 1）**
- 展示品类视觉惯例表中"标配程度：高"的条目。
- 确认：这些基准线图本品是否全部要做？有没有例外？

**D. 视觉手段候选库确认（来自竞品 Part 2）**
- 展示手段候选库摘要（不要全量，只展示与高优先级卖点匹配的手段）。
- 用户判断：有没有哪个手段明确不想用？有没有使用前提不满足、需要调整的？
- **不要求用户逐条选择，只需要否决或补充说明。**

**E. 竞品坑确认（来自竞品 Part 3）**
- 展示竞品结构性问题清单，说明本品规划中已标注的对应规避方向。
- 用户确认：有没有需要补充的避坑方向？

**F. 合规边界确认**
- 展示合规边界 Agent 标注的高风险项。
- 用户确认是否接受，有没有需要调整的表达范围。

**G. 不能碰的视觉方向**
- 用户是否有品牌、风格、受众相关的视觉禁区需要在此声明？

主控 Agent 输出：

- `03_STRATEGY/gate1_strategy_summary.md`

`gate1_strategy_summary.md` 必须包含：
- 用户确认的卖点优先级顺序（锁定，后续不再重复确认）
- 品类基准线确认结果
- 手段候选库中用户否决的条目
- 竞品坑补充说明
- 合规边界确认结论
- 视觉禁区声明

用户常用回复：

```text
Gate 1 通过，继续。
```

或：

```text
Gate 1 修改：
- 卖点优先级调整为：①XX ②XX ③XX
- 手段候选库中"前侧45°双人合成"不用，我们没有对应素材
- 补充视觉禁区：不用红色系配色
```

## Gate 2：图片序列确认

触发时机：主图、副图、A+、广告素材序列规划完成。

用户需要确认：

- 每张图的任务是否正确。
- 图2-3 是否承载最重要卖点。
- 是否遗漏必须讲的基准线卖点。
- 是否有图位需要增删或调换。

主控 Agent 输出：

- `04_IMAGE_SEQUENCE/gate2_sequence_summary.md`

用户常用回复：

```text
Gate 2 通过，继续生成 brief 和 prompt。
```

## Gate 3：最终候选确认

触发时机：候选图评估和迭代建议完成。

用户需要确认：

- 哪些图进入后期。
- 哪些图继续小修。
- 哪些图重构。
- 哪些图放弃。

主控 Agent 输出：

- `09_FINAL/final_candidate_summary.md`

用户常用回复：

```text
Gate 3 通过，整理最终交付。
```

## Gate 前允许提前询问的情况

只有以下情况可以提前问用户：

- 产品事实冲突，继续会导致产品画错。
- 合规高风险，继续会产生不应上线的 claim。
- 关键输入缺失，导致策略无法成立。
- 用户明确要求先停下来。

其他情况应先合理假设并标注在 `01_WORKFLOW/status.md` 和对应阶段文件中。
