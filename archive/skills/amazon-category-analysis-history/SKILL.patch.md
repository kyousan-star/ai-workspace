## [2026-06-10] 数据未全量分析，结论置信度存疑

- **场景**：数据集样本量较大时，skill 未分析全部样本就输出结论，且未告知用户实际分析了多少行数据
- **影响**：用户细看后发现结论基于部分数据，对 Go/No-Go 决策的可信度产生怀疑
- **根因**：skill 未强制要求 Claude 在输出前汇报样本覆盖情况；Claude 在数据量大时可能自行采样而不声明
- **修复方向**：在数据结构整理步骤（Step 3）加强制输出：分析前必须汇报"本次读取行数 / 总行数 / 覆盖率"；若覆盖率低于 80% 需明确告警，并询问用户是否继续
- **状态**：待写入 SKILL.md Step 3

## [2026-06-10] Codex 与 Claude Code 输出丰富度差异明显

- **场景**：同一份品类数据，在 Codex 和 Claude Code 分别跑同一 skill，结论丰富度和深度有明显差距
- **影响**：用户对哪个平台的结论更可信感到困惑；在 Claude Code 上得到的分析可能遗漏维度
- **根因**：两个平台的上下文处理机制和模型不同；skill 未针对平台差异做适配；部分分析逻辑可能依赖参数化知识，在不同模型上发挥不稳定
- **修复方向**：
  1. 把强依赖精确计算的模块（如评分卡、阈值判断）显式化为 if-then 规则，减少对模型自由发挥的依赖
  2. 在输出末尾加数据溯源 Footer（已在 v3 中有类似机制），让用户能核查结论来源
  3. 长期：建立 Golden Set（3-5 个标准案例 + 期望输出），跨平台对比时有基准参照
- **状态**：部分已在 v3 解决（数据溯源），评分卡显式化待处理
