# Amazon 品类洞察决策 Skill

## 定位与目标

本 Skill 用于对**已选定的 Amazon 品类**进行深度洞察分析，核心目标：

1. **判断品类可行性** - 这个品类值不值得进入？
2. **发现机会点** - 哪些细分赛道有机会？差异化方向在哪？
3. **评估进入门槛** - 需要多少资金？有什么壁垒？
4. **输出决策建议** - Go / Conditional Go / Hold / No-Go

---

## 使用说明

### 必填参数（开始前向用户确认）

| 参数 | 说明 | 示例 |
|------|------|------|
| `{品类名称}` | 英文品类名 | handheld massager |
| `{数据目录}` | Excel 原始文件所在路径 | ~/Documents/选品洞察/US/US handheld massager |
| `{价位段划分}` | 价格区间（用户可自定义） | 0-50, 50-100, 100-150, 150-200, >200 |
| `{最新月份}` | 数据中最新的月份 | 2026.02 |
| `{对比年份}` | 用于 YoY 对比的两个年份 | 2024 vs 2025 |
| `{核算币种}` | 所有利润测算统一使用的币种 | USD（默认）/ CNY / EUR |
| `{COGS来源}` | 采购成本获取方式，二选一：① **自行输入**（有工厂报价则直接告知）；② **查询估算**（Claude 通过 web-access 查 1688/Alibaba 参考价，结果标注「⚠ 估算」） | 自行输入 / 查询估算 |

> **COGS 提问规则**：启动分析时必须向用户确认以上两项。若选择自行输入，需提供各目标价位段对应的预估采购价（{核算币种}计）；若选择查询估算，Claude 将在执行模块 H 前自动查询并向用户确认后使用。

### 可选数据源（如有请提供）

| 数据类型 | 说明 | 用途 |
|---------|------|------|
| 关键词数据 | 搜索量、竞争度等 | 关键词与流量分析 |
| 评论数据 | Top产品评论 | 消费者洞察分析 |
| VOC报告 | 用户声音报告 | 痛点与需求挖掘 |

### 默认规则（除非用户另行指定）

- **新品定义**：生命周期 < 180 天的产品
- **未死亡 ASIN**：在最新月份仍出现在"产品详情"表且有库存的 ASIN
- **死亡定义**：销售从某个月开始到最新月份为 0
- **时间格式**：2024.10 和 2025.10 必须显示完整，不能显示为 2024.1 和 2025.1

### 时间范围计算规则（滚动24个月数据）

源数据通常为滚动24个月数据，时间范围需根据实际数据动态计算：

1. **识别时间范围**：读取所有Excel文件，识别数据覆盖的起止月份
   - 示例：数据覆盖 2024.03 - 2026.02
2. **确定对比年份**：
   - 取数据中包含的两个完整/部分年份作为对比年份
   - 示例：对比年份为 2024 vs 2025
3. **YoY 同期计算**：
   - 前一年时间范围：从数据起始月份到该年12月（如 2024.03-2024.12，共10个月）
   - 后一年同期范围：与前一年月份数一致（如 2025.03-2025.12，共10个月）
   - 确保两个年份的月份数相同，YoY 才具有可比性
4. **季度判断**：
   - 只有包含完整3个月的季度才参与季度分析
   - 不完整的季度单独标注或排除

> 执行前需向用户确认识别到的时间范围和对比规则

# 第一阶段：数据准备

> **⚡ 脚本加速**：Step 1（合并）和 Step 3（结构整理）可直接运行脚本，Step 2（打标）仍由 Claude 交互完成。
>
> ```bash
> # Step 1：合并
> python ~/.claude/skills/amazon-category-analysis/scripts/prepare_data.py \
>   --step merge \
>   --dir "{数据目录}" \
>   --category "{品类名称}"
>
> # Step 3：结构整理（打标完成后）
> python ~/.claude/skills/amazon-category-analysis/scripts/prepare_data.py \
>   --step structure \
>   --dir "{数据目录}" \
>   --category "{品类名称}" \
>   --price-tiers "{价位段划分}"
> ```

## Step 1：多表合并

**输入**：`{数据目录}` 下的多个结构一致的 Excel 文件
**输出**：`us {品类名称} 合并.xlsx`

操作：

1. **文件重命名**：名称最后面加"日期"，日期取自"产品"sheet 第一行中的信息。例如显示"2026-02-01至2026-03-01竞争报告"，则报告时间为 2026.02，文件重命名为 `20260304_3767551_2026.02`。

2. **统一结构**：
   - "产品详情"sheet：第一列改为"日期"（与报告时间一致，如"2026.02"），第二列全部插入"品类"列，填入 `{品类名称}`
   - "子体"sheet：插入第一列"日期"，第二列全部插入"品类"列，填入 `{品类名称}`

3. **去重**：通过日期识别重复的 Excel 表，如有重复，删除重复的保留一个。

4. **合并**：将去重后的所有 Excel 表合并为最终的"产品详情"sheet 和"子体"sheet，输出为 `us {品类名称} 合并.xlsx`。

> ⚠️ 注意：时间列中 2024.10 和 2025.10 要显示完整，不要显示为 2024.1 和 2025.1

---

## Step 2：结构化打标

**输入**：`us {品类名称} 合并.xlsx`
**输出**：`us {品类名称} 打标后.xlsx`

### 2.1 维度发现

从多个数据源自动提取候选维度：

| 数据源 | 提取方法 | 产出 |
|-------|---------|------|
| 产品标题 | 高频词聚类 + 语义归类 | 功能/材质/场景等候选维度 |
| 五点描述 | 关键词提取 | Spec相关候选维度（档位、电池、尺寸等） |
| 关键词数据（如有） | 修饰词分析 | 用户搜索视角的维度 |
| VOC/评论数据（如有） | 高频特征词提取 | 用户关注点视角的维度 |

### 2.2 维度框架整理

基于通用维度框架，结合品类特性整理维度列表：

| 维度类别 | 维度名称 | 说明 |
|---------|---------|------|
| **核心维度** | 产品类型 | 产品的本质分类（必打） |
| | 价位段 | 价格区间（必打） |
| **功能维度** | 核心功能 | 主要功能特性 |
| | 附加功能 | 增值功能 |
| **产品维度** | 动力方式 | 能源类型（充电/电池/插电） |
| | 材质 | 主要材质 |
| | 设计特点 | 外观/人体工学等 |
| **用户维度** | 目标人群 | 主要用户群体 |
| | 使用场景 | 使用环境 |
| | 适用部位/对象 | 针对的部位或对象 |
| **特殊标记** | 品类相关性 | 相关/边缘/不相关 |

### 2.3 用户确认维度

向用户展示候选维度列表，包括：
- 各维度名称及其候选值
- 用户可增加、删除、修改维度
- 确认后再执行打标

### 2.4 执行打标

1. **产品详情 sheet 打标**：根据"主图"、"产品名称"、"五点描述"信息对每个 ASIN 进行打标
   - 打标信息以**多列结构化输出**（每个维度一列），插入到"主图"和"产品名称"列之间
   - 信息来源优先级：标题 > 五点描述 > 主图
   - 无法识别的标记为"未知"
   - 如发现与品类不相关的产品（如情趣用品混入按摩器类目），在"品类相关性"列标注

2. **子体 sheet 打标**：根据"产品详情"sheet 中 "asin" 与 "子体" sheet 中 "变体asin" 的映射关系，同步打标到"子体"sheet

### 2.5 打标质量验证

1. 统计各维度"未知"占比
2. 如某维度"未知"占比 > 20%，提示用户：
   - 是否调整维度定义？
   - 是否人工补充部分数据？
3. 输出打标分布概览，供用户确认

### 2.6 产品类型分层判断（强制）

打标完成后，统计 `产品类型` 各值的销量占比：

- 若存在 **≥3 个子类型且每个占比 ≥10%**，则判定为**多形态品类**
  - 后续所有模块（A-H）必须以子类型为单位分别执行，**不得合并出结论**
  - 决策输出（评分卡、Go/No-Go）以子类型为单位独立给分
  - 向用户说明：「该品类包含 N 个主要子类型，将分别分析，以下为各子类型结果」
- 若子类型集中（1-2 个类型占比 >80%），则按整体品类正常执行

> ⚠️ 跳过此步骤是"一锅出"的根源。不同子类型的市场规模、竞争格局、进入门槛差异可能极大，合并分析会导致结论失真。

## Step 3：数据结构整理

**输入**：`us {品类名称} 打标后.xlsx`
**输出**：`us {品类名称} 数据结构整理.xlsx`

### 3.1 父体整理 sheet

根据"产品详情"表整理导出每个父体 ASIN by 季度的数据，列顺序：

```
品类, parentasin, 品牌, 店铺, [各打标维度列],
{前一年}Q1销量...{后一年}Q4销量, {前一年}Y销量, {后一年}Y销量, {后一年}Y同期销量,
{前一年}Q1销售额...{后一年}Q4销售额, {前一年}Y销售额, {后一年}Y销售额, {后一年}Y同期销售额,
{前一年}Q1ASP...{后一年}Q4ASP, {前一年}YASP, {后一年}YASP, {后一年}Y同期ASP
```

> **`店铺`字段说明**：来自"产品详情"表的`店铺`列（Amazon Store Name），作为卖家代理标识符使用——同一真实卖家可持有多个品牌但通常集中在少数店铺，`店铺`粒度比`品牌`更能反映实际控盘方。数据中无 seller_id 时统一用此字段。

### 3.2 子体整理 sheet

根据"子体"表整理导出每个子体 ASIN by 季度的数据，列顺序：

```
品类, asin, parentasin, [各打标维度列], 品牌, url, 上架时间, 上架天数,
{前一年}Q1销量...{后一年}Q4销量, {前一年}Y销量, {后一年}Y销量, {后一年}Y同期销量,
{前一年}Q1销售额...{后一年}Q4销售额, {前一年}Y销售额, {后一年}Y销售额, {后一年}Y同期销售额,
以及"产品详情"表中的 F,O,P,V-CE 列信息
```

> {后一年}Y 同期数据取值时间段根据"时间范围计算规则"动态确定，确保与{前一年}的月份数一致。

# 第二阶段：多维度洞察分析

**输入**：`us {品类名称} 数据结构整理.xlsx` + 关键词数据 + 评论/VOC数据（如有）
**输出**：`us {品类名称} 分析.xlsx`

所有分析均用**可视化图表 + 分析总结**的方式展示。
所有分析需要能 **by 打标维度筛选**，既能看到全貌也能下钻到具体的细分。

## 模块A：市场规模与趋势

基于"父体整理"和"子体整理"sheet 分析。

| 分析项 | 内容 | 输出 |
|-------|------|------|
| 大盘规模 | {对比年份} 年大盘年度销量、销售额、ASP 及 YoY | 表格+结论 |
| 季度趋势 | 销量、销售额、ASP 指标 by 季度的 YoY 变化 | 趋势图+分析 |
| **增长阶段判断** | 品类处于什么阶段？（快速增长/稳定增长/成熟/衰退） | 结论 |
| **季节性特征** | 是否有明显淡旺季？旺季在哪几个月？ | 图表+结论 |
| **细分增长点** | 哪些打标维度下的细分在增长/下滑？ | 细分趋势表 |

> 注意：后一年的年度数据取"同期"（与前一年覆盖的月份数一致），确保 YoY 更有可比性

---

## 模块B：竞争格局分析

基于"父体整理"和"子体整理"sheet 分析。

> **双层垄断视角**：品牌垄断度（基于`品牌`聚合）反映品牌认知壁垒；**店铺垄断系数**（基于`店铺`聚合）反映真实卖家控盘程度。两者结合才能识别"品牌看似分散、实为少数卖家多品牌操控"的隐性垄断陷阱。

| 分析项 | 内容 | 输出 |
|-------|------|------|
| 品牌份额 Top10（年度） | {对比年份} 年销量份额 Top10 品牌，份额、YoY | 表格+分析 |
| 品牌份额 Top10（季度） | by 季度的销量/销售额份额 Top10 品牌变化 | 趋势图+分析 |
| 品牌垄断度 | CR3/CR5/CR10 by 季度变化趋势（按`品牌`聚合） | 趋势图+结论 |
| **店铺垄断系数** | 按`店铺`字段聚合销量，计算前3名店铺销量占全部销量的比例（即卖家级 CR3）；同时输出 Top5 店铺及其旗下品牌数 | 系数值+趋势图 |
| **垄断类型四象限判断** | 对比品牌 CR3 与店铺 CR3，输出垄断类型标签和策略建议（见下表） | 类型标签+结论 |
| Top Parent ASIN 垄断度 | by 季度的 Top ASIN 集中度分析 | 表格+分析 |
| **头部玩家画像** | 对份额 >5% 的店铺，输出以下五项：① **ASIN迁移检测**：统计历史ASIN数量及首次上榜时间，若两个ASIN时间相邻且评论数差距<20%，标注「疑似迁移」；② **评论异常检测**：首次上榜评论数>200标注「疑刷量/Vine」，任意月份评论负增长标注「疑删评」；③ **打法分类**：按低价走量 / 评价壁垒 / 多SKU铺货 / Q4礼品爆发 / 新品冲榜 / 高价差异化归类；④ **多品牌控盘检测**：列出该店铺旗下所有品牌，若持有 ≥2 个品牌且合计份额 >20%，输出「⚠️ 多品牌控盘」警告；⑤ **竞品定位语义分析（v2新增）**：提取该店铺主力 ASIN（份额最高的 1-2 个）的 Title 前10词 + Bullets 高频修饰词，输出"定位摘要"（核心主打词、差异化话语、价格锚点、空白方向），格式见下表 | 每店铺一张标准卡片 |
| **新进入者机会** | 是否有新品牌/新店铺崛起？靠什么崛起的？ | 案例分析 |

**竞品定位语义摘要格式**（每个份额 >5% 店铺）：

| 字段 | 内容 |
|------|------|
| 店铺名 | - |
| 核心主打词 | 从 Title/Bullets 提取的最高频定位词（3-5个） |
| 差异化话语 | 他们强调什么独特价值（如"Professional grade / Silent motor"） |
| 价格锚点 | 主力 ASP 区间 |
| 空白方向 | 他们 **未覆盖** 的需求或规格（这是我们的差异化入口） |

> 所有竞品定位语义摘要汇总对比后，用于模块 E 的差异化方向判断——避免"我们的差异化其实已被头部覆盖"的决策跑偏。

**垄断类型四象限**（品牌 CR3 vs 店铺 CR3，阈值均以 40% 为界）：

| 品牌 CR3 | 店铺 CR3 | 类型标签 | 含义与策略 |
|---------|---------|---------|-----------|
| 低(<40%) | 低(<40%) | ✅ 分散市场 | 品牌和卖家都分散，真正可进入的蓝海，新品有机会 |
| 高(≥40%) | 低(<40%) | ⚠️ 品牌壁垒型 | 品牌认知强但卖家分散，需差异化定位或长尾切入 |

## 模块E：细分市场机会（新增"细分机会分析"sheet）

**前置条件**：模块 G 已完成，维度-需求对照表已就绪。

基于打标维度进行交叉分析，发现供需双侧均有支撑的蓝海细分。

| 分析项 | 内容 | 输出 |
|-------|------|------|
| 维度交叉矩阵 | 基于打标维度（如用途×价位×功能）做交叉分析 | 矩阵表 |
| 各细分市场规模 | 各交叉细分的销量、销售额、ASIN数量 | 汇总表 |
| **需求-供给四象限（v2新增）** | 将各细分按"需求强度（痛点严重度）× 供给稀缺度（活跃ASIN数/增速）"打入四象限，作为核心决策可视化 | 四象限散点图 |
| **蓝海细分推荐** | 3-5 个潜力细分赛道（仅从四象限"核心机会"区取），每条附 VOC 支撑强度评级 | 机会清单（见下） |

**需求-供给四象限矩阵（v2核心）**：

```
              供给少（活跃ASIN数低 且/或 增速慢）  |  供给多（活跃ASIN数高 且/或 增速快）
需求强         🎯 核心机会（优先进入）              |  🔴 红海（不进，已充分供给）
(痛点严重度高)  ─────────────────────────────────|──────────────────────────────────
需求弱         ⏳ 太早（等待需求成熟）              |  ⚪ 无意义（需求不足且拥挤）
(痛点严重度低)                                   |
```

**四象限定位规则**：

**需求强度轴（per-细分聚合方法）**：
1. 模块 G 已为每条痛点/需求建立了"维度映射"（如"续航短" → 动力方式维度）和"严重度得分"
2. 每个细分赛道由若干打标维度值的组合定义（如：动力方式=充电式 × 价位段=$30-60）
3. 该细分的**需求强度 = 所有映射到该细分核心维度的痛点严重度得分之和**
4. 所有细分的需求强度计算完毕后，取**中位数**为分界线：≥ 中位数 → "需求强"，< 中位数 → "需求弱"

> 示例：细分"充电式×$30-60"的核心维度是"动力方式"。模块G中映射到"动力方式"的痛点有"续航短（70.5分）"和"充电慢（32分）"，则该细分需求强度 = 102.5分。

**供给稀缺度轴**：
- 该细分活跃 ASIN 数 **低于** 品类均值，且 YoY 增速 **低于** 品类均值 → **供给少**
- 两项中只满足一项 → 供给适中（在散点图上落在轴线附近，仍可标注但不优先）
- 两项均高于均值 → **供给多**

> 只有落在"核心机会"象限的细分才进入蓝海推荐清单。其他象限明确标注放弃原因，防止会议讨论时反复纠结已排除的方向。

**蓝海细分推荐输出格式**：

| 细分赛道 | 象限位置 | 市场规模 | YoY增长 | 活跃ASIN数 | 痛点严重度 | 店铺垄断风险 | VOC关键信号 | 竞品空白方向 |
|---------|---------|---------|---------|-----------|----------|------------|------------|------------|
| - | 核心机会 | - | - | - | 高/中 | 低/中/高⚠️ | 对应痛点词 | 来自模块B语义分析 |

**店铺垄断风险评级规则**（该细分内前3名店铺销量占比）：
- **低**：前3名店铺占比 < 30%，新进入者有真实曝光机会
- **中**：30%-50%，可进入但需在差异化上有明显壁垒
- **高⚠️**：> 50%，少数卖家控盘，即使品牌垄断度低也属于隐性陷阱，不得进入最终推荐清单

**VOC 支撑强度评级规则**：
- **强**：模块 G 中有直接对应的未满足需求条目，且该条目痛点严重度得分排名前 30%
- **弱**：G 中有间接相关信号，但严重度得分排名在 30%-70% 区间，或仅为需求方向暗示
- **无数据**：无 VOC 数据，整条机会点标注「⚠ 供给侧推断，需求未验证」

> 店铺垄断风险"高⚠️"的细分直接排除，不进最终推荐清单；评级"无数据"的机会点仅作备选，差异化空间维度得分不得 ≥ 7 分。

### H.2 单品经济模型（利润 Gate）

**针对模块 E 推荐的每个细分机会分别计算**，所有金额统一使用 `{核算币种}`。

若 `{COGS来源}` = 查询估算，在此步骤前通过 web-access skill 查询 1688/Alibaba 参考价，向用户确认后使用。

**计算结构**：

```
目标 ASP（来自该细分的数据均值）
  - Amazon 佣金        = ASP × 15%（标准，特殊品类按实际费率）
  - FBA 配送费         = 按产品尺寸/重量估算（参考当前 FBA 费率表）
  - PPC 广告费         = ASP × 15%（无关键词数据时用行业均值；有数据则用实际 CPC × 预估转化率）
  - COGS（采购成本）   = {用户输入} 或 {查询估算值，标注⚠}
  ─────────────────────────────────────
  = 单品净利润（{核算币种}）
  = 净利润率（%）
```

**利润 Gate 判定**：

| 净利润率 | 判定 | 对后续决策的影响 |
|---------|------|----------------|
| ≥ 20% | 通过 | 正常评分，进入机会点清单 |
| 10%-19% | 偏紧 | 进入门槛维度降 2 分；机会点清单标注「利润空间偏紧，需控 COGS 或提 ASP」 |
| < 10% | **Gate 否决** | 无论其他维度得分多高，该细分最终决策强制降为 Hold 或 No-Go；在决策摘要中说明原因 |

> 利润 Gate 是独立于五维度评分卡的硬约束，不参与加权计算，但可直接否决结论。
