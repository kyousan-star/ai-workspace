# Amazon PPC 广告智能优化系统 (US站)

> 完整工作流: 数据解析 → 指标量化分层 → 矩阵组合诊断 → 竞价建议 → 策略匹配 → 打法SOP → 动作日志(PDCA闭环)

## 使用前必读（每次分析的三个必要输入）

任何诊断动作之前，先向用户确认三件事，缺一不可：

1. **毛利率**（= 盈亏ACOS）: 所有 ACOS/ROAS 判断的基准。公式 `毛利率 = (售价×(1-佣金率) - 产品成本 - FBA费) / 售价`
2. **生命周期阶段** `launch / growth / stable`: 决定否词门槛和 ACOS 容忍度（新品期目标是收录+排名+单量，不是 ACOS，见下文"新品期模式"）
3. **核心保护词根**: 产品的核心大词（如 2-4 个词根），任何情况下不进入否定候选

所有阈值的唯一事实源是 `ppc_config.json`（毛利驱动、分阶段），诊断规则的唯一事实源是 `references/diagnosis_rules.json`。修改阈值改 config，不要改代码或文档里的数字。

## 1. 数据解析引擎

当用户上传亚马逊广告后台的数据文件时，执行 `scripts/ad_data_parser.py` 进行自动解析。

### 支持的报告类型

| 报告名称 | 英文名 | 关键字段 |
|----------|--------|----------|
| 搜索词报告 | Search Term Report | Customer Search Term, Impressions, Clicks, Spend, Sales, Orders |
| 投放报告 | Targeting Report | Targeting, Match Type, Impressions, Clicks, Orders |
| 广告位报告 | Placement Report | Placement, Impressions, Clicks, Spend, Sales |
| 已购买商品报告 | Purchased Product Report | Purchased ASIN, Advertised ASIN |
| 广告活动报告 | Campaign Report | Campaign Name, Budget, Impressions, Clicks |
| Rufus提示词报告 | Rufus Report | AI Prompt, Clicks |

### 数据解析流程

```
Step 1: 识别文件格式 → CSV/XLSX/TSV自动检测编码和分隔符
Step 2: 字段映射 → 中英文字段名自动匹配标准字段
Step 3: 数据清洗 → 去除空行/异常值/统一货币格式/日期标准化
Step 4: 衍生指标计算:
  - CTR = Clicks / Impressions
  - CVR = Orders / Clicks  
  - CPC = Spend / Clicks
  - ACOS = Spend / Sales
  - CPA = Spend / Orders
  - ROAS = Sales / Spend
  - TACoS = Spend / Total_Sales (如有总销售额)
Step 5: 指标分层标记 → 对每条数据打上分层标签
Step 6: 矩阵诊断 → 对每种组合输出诊断结果
Step 7: 汇总报告 → 生成分析结果和优化建议
```

运行方式:
```bash
python scripts/ad_data_parser.py <广告数据文件> \
  --margin 0.35 \            # 必传: 产品毛利率(=盈亏ACOS)
  --price 44.99 \            # 建议传: 客单价，传入后输出每个词的具体建议竞价
  --phase launch \           # 必传: launch(新品1-2月)/growth(2-4月)/stable(5月+)
  --core-terms "词根1,词根2" \ # 建议传: 核心保护词根，永不否定
  --attribution 7            # SP报表默认7天归因；SB/SD报表用14
```

输出: Excel(完整分析/否定候选/放大候选/上期动作核验/数据概览) + `ad_actions_log.json`(PDCA动作日志，自动写在数据文件同目录)。

### 竞价建议公式

```
建议竞价 = 盈亏ACOS × 客单价 × 平滑CVR × phase系数
平滑CVR = (该词订单 + k×整体CVR) / (该词点击 + k)   # k=10，贝叶斯平滑防小样本失真
phase系数: launch 1.3 / growth 1.1 / stable 1.0    # 新品期为抢位适度溢价
单次调价幅度限制在当前CPC的±50%，防止数据抖动导致激进调价
```

诊断输出必须给出具体数字（"提高竞价至$1.24"），不允许只说"提高竞价"。

### 新品期模式 (--phase launch)

上架 1-2 个月内的产品必须用 launch 模式，与稳定期的区别：

| 项目 | launch | stable | 原因 |
|------|--------|--------|------|
| 否定门槛 | ≥20次点击 或 花费≥3×盈亏CPA | ≥10次点击 | 新品CVR天然偏低，过早否词会砍死潜力词 |
| ACOS容忍 | 盈亏线×2.0 | 盈亏线×1.0 | 新品期买收录买排名，亏损是投资 |
| 目标函数 | 收录+排名+单量 | 利润 | 不要用ACOS高低评判新品期广告 |
| 竞价系数 | ×1.3 | ×1.0 | 抢首页坑位积累权重 |

新品期报告结构调整: 把"关键词收录数/核心词排名变化/订单趋势"放在ACOS之前汇报。

### PDCA 闭环（效益复利的关键）

每次运行 parser 会自动: ①核验上一轮推荐动作的实际效果（否定是否止血、放大是否增单），输出"上期动作核验"sheet；②把本轮推荐动作写入 `ad_actions_log.json`。**每轮分析必须先看核验结果再给新建议**——上轮放大的词没增单，先找原因（坑位没到位/竞价没执行/Listing问题），不要盲目叠加新动作。

## 2. 利润核算与指标量化

### 盈亏ACOS计算

在做任何广告诊断前，必须先确定产品的盈亏线:

```
盈亏ACOS = (售价 - 产品成本 - FBA费 - 佣金) / 售价 × 100%
         = 毛利率

核心公式:
  毛利 = 售价 × (1 - 佣金率) - 产品成本 - FBA费
  盈亏ACOS = 毛利 / 售价 × 100%
  CPA盈亏线 = 毛利 (每单能承受的最大广告花费)
  TACoS = 广告花费 / SKU总销售额 × 100%
  ACoTs = ACOS × ASoTs (广告花费/SKU总销售额)
  ASoTs = 广告订单 / SKU总订单 (广告销量占比)
```

### 四大指标量化分层标准

**曝光量分层** (基于投放对象类型动态调整):

| 档位 | 关键词广告阈值 | ASIN广告阈值 | 分层逻辑 |
|------|--------------|-------------|---------|
| 无曝光 | = 0 | = 0 | 完全无展示 |
| 低曝光 | < 500 | < 200 | 低于类目中位数50% |
| 中曝光 | 500-5000 | 200-2000 | 围绕类目中位数 |
| 高曝光 | > 5000 | > 2000 | 高于类目中位数200% |

**点击次数分层**:

| 档位 | 阈值 | 说明 |
|------|------|------|
| 无点击 | = 0 | 有曝光无点击 |
| 低点击 | 1-5次 | 数据不足以判断 |
| 中点击 | 6-20次 | 有一定参考价值 |
| 高点击 | > 20次 | 数据充分可做判断 |

**点击率CTR分层** (辅助指标):

| 档位 | 阈值 | 说明 |
|------|------|------|
| 低CTR | < 0.2% | 主图/标题/匹配度问题 |
| 中CTR | 0.2%-0.5% | 行业平均水平 |
| 高CTR | > 0.5% | 表现优异 |

**转化率CVR(广告订单)分层**:

| 档位 | 阈值 | 说明 |
|------|------|------|
| 0转化 | = 0 | 完全无成交 |
| 低转化 | < 5% | 低于行业均值 |
| 中转化 | 5%-15% | 行业平均水平 |
| 高转化 | > 15% | 表现优异 |

**ACOS分层** (基于盈亏ACOS动态调整):

| 档位 | 阈值 | 说明 |
|------|------|------|
| 极佳 | < 盈亏ACOS×0.5 | 利润丰厚 |
| 良好 | 盈亏ACOS×0.5 ~ 盈亏ACOS | 盈利状态 |
| 偏高 | 盈亏ACOS ~ 盈亏ACOS×1.5 | 微利或微亏 |
| 过高 | > 盈亏ACOS×1.5 | 严重亏损 |

> 以上表格仅为速查展示。**生效阈值以 `ppc_config.json` 为准**（含 launch/growth/stable 三档否词门槛与ACOS容忍系数），改阈值只改 config。

## 5. 广告类型与结构

### 7种广告类型

```
亚马逊广告
├── SP 商品推广 (建议预算占比60%)
│   ├── 自动广告
│   │   ├── 紧密匹配 Close Match (基于关键词)
│   │   ├── 宽泛匹配 Loose Match (基于关键词)
│   │   ├── 同类商品 Substitutes (基于商品)
│   │   └── 关联商品 Complements (基于商品)
│   └── 手动广告
│       ├── 关键词广告 (广泛/词组/精确)
│       ├── 类目定位广告 (品类投放+Refine)
│       └── ASIN定位广告 (捡漏/优势/互补/闭合)
├── SB 品牌推广 (建议预算占比20%)
│   ├── 商品集合广告
│   ├── 品牌旗舰店子页面广告
│   └── 视频广告 SBV
├── SD 展示型推广 (建议预算占比20%)
│   ├── 商品投放 (竞品页面定位)
│   └── 再营销浏览定向 Retargeting
└── DSP 程序化广告
    └── 需客户经理开通
```

### 竞价策略选择

| 策略 | 适用场景 | 效果 |
|------|---------|------|
| 固定竞价 | 新品确保曝光/测试竞价效果 | 稳定曝光,转化率可能较低 |
| 动态竞价-提高和降低 | 成长期/追求转化 | 高转化场景自动加价,低转化自动减价 |
| 动态竞价-只降低 | 控制成本/不主推产品/清库存 | 降低广告费,抑制曝光 |

### 广告位竞价增幅

| 广告类型 | 搜索结果首页顶部 | 其余位置 |
|----------|-----------------|---------|
| SP商品推广 | 不超过基础竞价100% | 不超过50% |
| SB品牌推广 | 无 | 无 |
| SD展示型 | 不超过300% | - |

### 产品生命周期广告节奏

| 阶段 | 周期 | 流量策略 | 广告占比目标 |
|------|------|---------|------------|
| 新品期 | 1-2月 | 高相关精准词主攻,广告→自然过渡 | 70-80% |
| 成长期 | 2-4月 | 扩展中相关词,目标前3页 | 50-60% |
| 稳定期 | 5-6月+ | 扩低相关词+类目+竞品流量 | 30-40% |
| 衰退期 | 视情况 | 铺开入口,保本CPC通投 | 降至最低 |

## 7. 影响广告的13大因素

诊断时必须考虑多维度因素，不能只从单一广告维度衡量：

### 产品因素 (影响CTR和CVR)
1. 首图质量/附图/A+/视频质量
2. 产品标题 (简洁明了/核心词突出)
3. 产品价格 (相对竞品)
4. 评分Rating和评论Review数量
5. 促销折扣/Coupon/秒杀标志(LD/BD/DOTD)
6. 亚马逊标签 (Best Seller/Amazon Choice/Hot New Release)
7. 显示要素 (多颜色尺码/划线价/30天最低价)
8. 产品亮点差异化
9. 配送方式和时效

### 广告因素 (影响曝光和CTR)
10. 广告竞价 (基础竞价/广告位竞价/竞价策略)
11. 匹配方式和否词策略
12. 广告位 (首页顶部>其余>商品页面)
13. 广告预算和投放时长

### 其他因素
- 购物车(BuyBox)赢得率
- 库存状态(是否低库存)
- 跟卖情况
- 账号绩效安全
- Listing是否有在售信息
- 品牌知名度
- 广告位周边竞品情况

## 8. 执行指南

### 收到广告数据文件时的完整流程

```
1. 向用户确认三个必要输入: 毛利率 / 生命周期阶段 / 核心保护词根
   (缺客单价则一并问，用于竞价建议)
2. 运行数据解析 → scripts/ad_data_parser.py --margin X --price Y --phase Z --core-terms "..."
3. 先看"上期动作核验"结果 → 上轮动作没生效的先找原因
4. 加载诊断规则 → references/diagnosis_rules.md (如需场景详细规则)
5. 加载打法SOP → references/ad_plays_sop.md (推荐打法必须带风险等级)
6. 生成分析报告 → 新品期把收录/排名/单量放在ACOS之前汇报
7. 生成广告日志 → scripts/ad_log_generator.py --break-even X
```

### 收到广告诊断咨询时

```
1. 确认产品基本信息 (售价/成本/FBA费/佣金率) → 计算盈亏ACOS = 毛利率
2. 确认生命周期阶段 → launch/growth/stable (阈值差别很大，不能跳过)
3. 确认广告数据 (曝光/点击/订单/ACOS) → 判定矩阵组合
4. 读取 references/diagnosis_rules.md 查找对应场景
5. 输出: 问题分析 + 广告目的 + 广告策略(含具体建议竞价数字) + 预期效果
6. 推荐具体打法SOP(带风险等级)
```

### 关键词优化逻辑

```
关键词分层:
  流量分级: 一级(大词) / 二级(中词) / 三级(长尾词)
  相关性: 强(50-100%) / 中(20-50%) / 弱(1-20%) / 不相关(0-1%)
  
关键词来源:
  ① 10个核心竞品ASIN多产品流量对比 (SIF)
  ② 20个竞品ASIN拓展流量词 (卖家精灵)
  ③ 50-100个类目竞品SIF关键词收集
  ④ 商机探测器关键词份额分析
  ⑤ 竞争格局自然流量份额分布

选词逻辑(算法):
  综合评分 = 商业价值×权重 + 广告选词×权重 + 点击转化率×权重
  
  投放优先级:
    核心投放(≥80分): 日预算25-35%
    重点投放(65-79): 日预算15-25%
    标准投放(50-64): 日预算10-15%
    测试投放(35-49): 日预算5-10%
    低价测试(20-34): 日预算<5%
    暂不投放(<20): 不设预算
```
