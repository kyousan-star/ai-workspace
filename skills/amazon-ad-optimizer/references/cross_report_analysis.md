# 多报表交叉分析（原 amazon-ads-analysis 并入）

> 广告数据只看广告报表会失真：广告的真正回报包含它拉动的自然单。本文档覆盖跨报表交叉分析——广告×自然流量、SQP内卷、分时段。字段详解见 `report_fields.md`，阈值见 `../ppc_config.json`。

## 一、报表清单与分析价值

### 必要报表

| 报表 | 来源 | 核心字段 | 分析价值 |
|------|------|----------|----------|
| **流量表现** | Business Report | Sessions, CVR, Buy Box% | 整体流量、转化率、跟卖检测 |
| **SP投放商品** | Advertised Product | Spend, Sales, Orders | 广告活动效率、趋势分析 |
| **SP投放报告** | Targeting Report | Match Type, Impression Share | 匹配类型效率、竞价机会 |
| **广告搜索词** | Search Terms | Customer Search Term | 高效词/否定词挖掘 |
| **购物时间** | Campaign Hourly | Hour, Day of Week | 时段效率分析 |

### 可选报表

| 报表 | 分析价值 |
|------|----------|
| **销售数据** | 真实盈亏（含广告后净利润） |
| **SQP数据** | 自然订单，内卷分析 |
| **广告位报告** | TOS/ROS/PP效率对比 |
| **预算报告** | 预算消耗率/受限时长（ROADMAP #6 依赖） |

## 二、快读分析流程（5步法）

```
第一步：全局评估
├── 产品是否盈利？（销售数据，看毛利总额-广告花费）
├── Buy Box是否正常？（<95%查跟卖）
├── 整体CVR如何？
└── 判断：问题在广告还是产品本身？

第二步：广告效率诊断
├── 各广告活动 利润贡献($) 和 ACOS
├── 匹配类型效率：EXACT vs BROAD vs AUTO
├── 展示份额：高ROAS + 低份额 = 加投机会
└── 广告位效率：TOS vs ROS vs PP

第三步：搜索词分析
├── 高效词筛选：ROAS > 盈亏ROAS 且有订单，按利润贡献排序
├── 零转化词：达到当前生命周期否定门槛（见 ppc_config.json）
├── 出单未投放词：自动/广泛跑出的出单词，未开精准的立即收割
└── 意图分类：品牌词/品类词/竞品词/长尾词

第四步：时间与趋势
├── 周环比：CVR是否爬坡、CPC是否下降（新品期健康度）
├── 时段效率：哪些小时ROAS高/低（PST口径）
└── 低效时段浪费：花费占比 vs 订单占比

第五步：输出建议（全部按利润贡献美元排序）
├── 否定清单（关键词/ASIN分开，附理由）
├── 收割清单（出单未投放词+建议竞价）
├── 竞价调整（具体数字）
└── 预算重分配建议
```

## 三、交叉分析矩阵

| 分析目的 | 报表组合 | 计算方法 |
|----------|----------|----------|
| **广告流量占比** | 流量表现 + SP投放商品 | Ad Clicks / Total Sessions |
| **广告订单占比** | 流量表现 + SP投放商品 | Ad Orders / Total Orders |
| **TACoS** | 流量表现 + SP投放商品 | Ad Spend / Total Sales |
| **内卷检测** | 广告搜索词 + SQP | 同词广告&自然都有单 |
| **展示份额机会** | SP投放报告 | ROAS>盈亏ROAS 且 Share<5% |
| **真实盈亏** | 销售数据 + SP投放商品 | 毛利总额 vs 广告花费 |

## 四、关键公式

```python
# 盈亏基准（务必用产品真实毛利率）
盈亏ACOS = 毛利率 = (售价×(1-佣金率) - 产品成本 - FBA费) / 售价
盈亏ROAS = 1 / 盈亏ACOS
盈亏CPA = 毛利率 × 售价

# 收益最大化的核心指标（美元优先于比率）
利润贡献 = 销售额 × 毛利率 - 广告花费     # 词级/活动级都适用
TACoS = 广告花费 / 总销售额               # 广告健康度北极星
自然订单占比 = (总订单 - 广告订单) / 总订单  # 上升=广告飞轮生效

# 流量结构
广告流量占比 = Ad_Clicks / Total_Sessions
广告订单占比 = Ad_Orders / Total_Orders

# 内卷分析
内卷花费 = 广告和自然都有订单的词的花费
内卷比例 = 内卷花费 / 总广告花费
```

## 五、US市场口径

- 时区: 广告报表默认 PST；时段分析以 PST 解读，跨美东美西4小时时差
- 归因窗口: SP报表7天，SB/SD报表14天，交叉分析必须统一口径
- 高效时段: 通常 18:00-22:00 当地时间，以自己数据为准
- 大促节奏: Prime Day(7月)、返校季(8月)、黑五网一(11月)、圣诞(12月)，大促前2-3周CPC上涨

## 六、常见问题

**Q1: ACOS高但产品盈利？**
看毛利总额 − 广告总花费，以及 TACoS 趋势。自然订单占比高时，高 ACOS 完全可接受；新品期 ACOS 容忍到盈亏线×2（见 ppc_config.json phases）。

**Q2: 如何判断内卷？**
广告搜索词 × SQP 交叉：同一词两边都有订单 = 内卷，可考虑降低该词广告投入。注意新品期不要急于降投——排名还没坐稳时"内卷"其实是防守。

**Q3: 展示份额很低正常吗？**
<5%常见，看效率：高ROAS+低份额=加投机会；低ROAS+低份额=正常。

```markdown

## 一、执行摘要

### 1.2 流量与效率指标
### 1.3 关键发现（3-5条）

## 二、上期动作核验   ← 必须在新建议之前
## 三、广告活动分析（按利润贡献排序）
## 四、匹配类型/广告位效率

```
