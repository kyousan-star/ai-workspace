# 盈亏平衡测算 + 推广沙盘预演（GO 后附加模块）

> **来源**：自 amazon-product-dev skill（已 deprecated）的 D4/D5 维度迁入，2026-07-02。
> **使用时机**：Step 3.5 评分落在 GO / CONDITIONAL GO 时，在报告"投入产出测算"章节之后追加本模块，回答"从 0 推到稳态要花多少钱、多久回本"。HOLD / NO-GO 不输出。
> **口径纪律**：单位经济数字沿用 Step 3 的测算（自家 BOM 口径），禁止在本模块另起一套估算；CPC/ACOS 用 Step 1.2 关键词数据的实际值，无数据时用 `~/ai-workspace/sops/amazon-decision-thresholds.md` 的行业均值并标注⚠。

## 一、盈亏平衡测算（三阶段）

**阶段1：新品推广期（Month 1-3）**
```
月销量假设：50-100单（或按 amazon-category-analysis 排名分层"追赶层"中位月销，若有面板终验数据）
广告花费：ACOS 80-120%（新品高ACOS正常）
自然单占比：20-30%
月度盈亏 = 总收入 - 总成本 - 广告花费 → 预计亏损 $XXX/月
```

**阶段2：成长期（Month 4-6）**
```
月销量假设：150-300单（或"腰部层"中位月销）
广告花费：ACOS 40-60%
自然单占比：40-50%
月度盈亏 ≈ 盈亏平衡
```

**阶段3：稳态期（Month 7+）**
```
月销量假设：300-500+单
广告花费：ACOS 20-30%
自然单占比：60-70%
月度盈亏 = 稳定盈利
```

**关键公式**：
```
盈亏平衡月销量 = 固定成本 / (单位贡献利润 - 单位广告成本)
回本周期 = 总推广投入 / 月度净利润（稳态期）
```

**输出**：三阶段现金流表 + 累计投入峰值（这个数进启动现金全口径，对照可用弹药——超出即触发资金排除项，见 blue-ocean-finder feasibility_gate A4）。

## 二、推广沙盘预演（从 0 到稳态）

**Phase 0：Pre-Launch（上架前2-4周）**
- 产品到仓确认；Listing 优化完毕（标题/图片/A+/视频）
- 关键词清单准备（直接用 Step 1.2 的关键词矩阵）
- Vine 计划申请（或站外测评渠道）；预算分配计划

**Phase 1：Launch（Week 1-4）**
- 开启自动广告（发现流量）+ 手动精准/短语匹配（核心词）
- 配合优惠券/折扣冲初始订单
- 目标：日均5-10单，积累15-30个Review；预算 $/天按盈亏测算阶段1填

**Phase 2：Ramp-Up（Month 2-3）**
- 优化广告结构（否定词/竞价调整）；扩展长尾词/竞品词；开启SB品牌广告
- 目标：日均10-20单，Review 50+

**Phase 3：Scale（Month 4-6）**
- 分时段/分位置竞价；开启SD展示广告；站外引流
- 目标：日均20-40单，进入品类 Top50（或面板终验给出的"腰部层"排名）

**Phase 4：Profit（Month 7+）**
- 降广告依赖提自然占比；ACOS 优化到目标水平；考虑变体扩展
- 目标：TACOS < 12%，稳定盈利

**输出**：各 Phase 的目标单量/Review/预算表，目标锚点优先取 amazon-category-analysis 面板终验的排名分层数据（追赶层→腰部层），无面板数据时用 Sorftime Top100 分布估算并标注⚠。
