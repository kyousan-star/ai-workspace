# GO 前审计清单（决策点② 之前强制跑，10-20 分钟）

> 来源：overhead / teleprompter 两个实测项目的事后审计。两案的翻盘性错误（变体错觉推翻 GO、假 YoY、"零App"主张不自洽、分数不一致）全部是这类审计翻出来的，没有一个是流程自动拦下的——所以把审计制度化，放在拍板前而不是压货后。
> 执行方式：逐项核对报告原文 + 必要时实拉 Sorftime 复核（标 🔧 的项需要实拉，共 3-5 次调用）。任一项 ❌ → 修正报告、机械重打分（铁律#8），再进决策点②。

## A. 数据口径（两案的头号翻车区）

- [ ] 🔧 **变体核查**：每个对标/标杆 ASIN 跑过 `product_variations`？销量用子体真实口径、评论壁垒用买家页面所见（家族合并）口径？上架时间查了（"新品"可能是三年老链接）？（铁律#7）
- [ ] 🔧 **家族销量陷阱**：product_detail/product_trend 的月销按家族口径理解了吗？有没有把家族销量记到低评论子体头上造出"低评论高销"？
- [ ] **分桶值**：同页多产品月销完全相同的，当"<量级"读了还是当精确值加总了？
- [ ] **环比 vs 同比**：所有"暴跌/暴涨"结论对过去年同月吗？（overhead"6月-46%"实为季节谷、YoY +82%）
- [ ] **促销拆分**：销量锚有没有混入 Prime Day/Deal 尖峰？

## B. 需求证据

- [ ] **噪音 YoY**：有没有对多月为 0 / 中位 <1k 的词算同比？（scanner ❓ 词进了结论吗）
- [ ] 🔧 **词形迁移**：核心依据若是垂直起量的新词，词族相邻词（父词/同义词）对照做了吗？总盘是平移还是净增？
- [ ] **主锚强度**：需求判断的主锚是 ≥13 个月、各月同比可核的词，还是一个短历史长尾词？

## C. 竞争证据

- [ ] 🔧 **首页普查**：竞争/空白判断有 `keyword_search_results` 自然位首页普查支撑吗？（铁律#10）
- [ ] **空白举证**："空白/唯一/无人做"结论出自全量口径（首页/完整Top100）还是精选子样本？
- [ ] **节点碎片化**：对标品都在被分析的节点里吗？跨节点散布时天花板用首页普查口径了吗？

## D. 产品与主张

- [ ] **主张自洽走查**：核心卖点放进用户完整使用流程（开箱→安装→完成任务）走过一遍吗？每一步字面成立吗？（铁律#12，"零App"教训）
- [ ] **痛点同段**：差评弹药和目标产品在同一价格段吗？跨段引用标注折扣了吗？
- [ ] **观察闸门 vs 季节窗口**：若结论挂了"等 X 月数据再定"，它和"赶 X 季旺季"互斥吗？互斥的话报告点破了吗？（overhead"等7-8月"与"赶Q4"死锁教训）
- [ ] **举证材料自检**：被引为正向证据（低壁垒/低成本切入）的标杆 ASIN，是否恰好属于报告同时建议规避的风险类别（专利/合规/形态）？举证材料本身踩雷会不打自招地推翻它想支撑的结论。（2026-07-20 金标准回归实测：Sonnet 独立发现 overhead v3 用来证明"低评论高销、切入成本低"的磁吸款 B0GDWFCQGV，正是报告别处建议规避的磁吸专利风险形态——这条 v4 本身都没点破）

## E. 经济与评分

- [ ] **毛利口径**：用的是自家 BOM/1688 报价还是工具对在位者的估算？Sorftime"毛利率"不含 COGS，进决策了吗？（铁律#9）
- [ ] **CVR 生死线模型**：广告经济模型做了吗（盈亏平衡 CVR、打平自然占比、到 50 评投入）？带电/易退品类计退货率了吗？
- [ ] **资金闸门**：全口径现金（首批货按年1锚量 + 广告 + Vine + 周转）≤ 可用弹药？"条件性 GO"有没有绕过资金排除项？
- [ ] **TAM 诚实性**：三角验证法与法差 >3 倍还叫"收敛"吗？
- [ ] **分数单一出处**：全文只有一个分数值？每轮补验证后机械重算过、没有"挂起不入分"？（铁律#8）
- [ ] **B 段自检表**：报告末尾的 feasibility_gate B 段覆盖表齐全？未覆盖维度按保守值打分了吗？

## F. 交付

- [ ] **溯源表**：每个关键数字能回溯到具体调用/脚本/明确标注的估算？
- [ ] **版本主权**：本版定稿后，旧版标 superseded、RESUME/种子池同步了吗？
