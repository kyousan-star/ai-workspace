# 可行性硬验证闸门（feasibility gate）

> 缝隙"存在"（段1需求涨+段2竞争浅）≠ "能做出赚钱 SKU"。
> 本清单固化自 overhead 项目的 9 项提质教训——正是这些维度原来没查，才让 overhead 从 GO 7.5 虚高、补验证后降到 6.7。
> 分两段用：**A 快筛**（blue-ocean 出 shortlist 时跑，10-30 分钟/候选，挡掉做不了的）+ **B 交接清单**（交 zach 时带上，zach 深挖必须覆盖，是 Go/No-Go 打分的输入）。

---

## A · blue-ocean 阶段快筛（交 zach 前，一票否决优先）

### A1 IP 快筛（一票否决，最先做）
- 用 WebSearch/WebFetch 查 USPTO / Google Patents：核心**形态 + 机构 + 差异化卖点**（尤其磁吸/特殊结构）有无生效的**实用新型(utility)**专利。
- 看两点：① 权利要求是否覆盖我方核心组合；② 权利人是否积极维权方（大摄影/配件品牌如 Videndum/Manfrotto/JOBY、SmallRig 等会走 Amazon APEX，下架快）。
- 判定：明显踩雷 + 维权方 → **换形态或换方案规避**，否则弃。外观专利(design)风险低（差异化外形即可）。
- 实证：overhead 磁吸夹踩 Videndum US11796126B2（utility，54mm环磁+旋转夹臂）→ 首发改纯机械夹持款绕开。

### A2 稳态销量（拆促销）
- 竞品销量看**多月 product_trend + 价格 trend**，不看单日/单月。价格回弹→销量跌 = 促销依赖，稳态取非促销月。
- 实证：overhead 6月销量 -46% 伴随均价回 $49.99；"137单/天"是 Prime Day 尖峰，稳态 50-70/天。

### A3 形态天花板（不是搜索池）
- 我方能承接的**形态**在节点里做多少量？节点头部是别的形态就按我方形态段封顶。
- 实证：overhead 节点量被 $16 廉价夹灯占，rig 形态天花板仅 2.5-4k/月，非搜索池 2万。

### A4 启动现金量级（粗算）
- 粗算"评论 0→50 阶段广告烧钱"：`每单广告成本 = 混合CPC ÷ 广告CVR`；`到50评所需订单 ≈ 700（Vine+催评）~3300`。
- 若结果把标的从"试水"变"重投入"（如 >账户单标的弹药），标注真实量级，别用"低风险试水"误导。
- 实证：overhead 烧到50评需广告 $10-29k、净亏现金 $0-11k → 定性从"试水"改"中投入"。

**A 段任一红灯 → 该候选降级/换形态/弃，不进 zach。**

### A5 跨平台交叉验证（可选，非一票否决）

> 定位：**风险审计，不是发现工具**。只在 A1-A4 都过关的候选上跑，不要在 P0 种子阶段就跑——实测(magic arm案例)证明 TikTok Shop 的销量/内容热度节奏跟 Amazon 搜索**同步而非领先**，不是能提前抓到红利的哨兵；放在 P0 只会多花 6-8 次工具调用换不到领先信号。

**用 Sorftime TikTok 工具跑（US站，约6-8次调用/候选）**：
1. `tikto_category_search_from_name`（目标形态相关词）→ 找对应类目节点
2. `tiktok_category_report`（该节点）→ 看月销量环比：是否与 Amazon 判断方向一致（都在跌/都在涨），及头部产品是不是我方目标形态（常见坑：类目在涨但拉动的是别的子形态，同铁律#3）
3. `tiktok_similar_product`（目标产品名）→ 找同形态在 TikTok Shop 的对应 SKU
4. `tiktok_product_trend`（找到的 productId）→ 看周度销量/新增视频/新增达人数时序，判断见顶/冷却时间点是否与 Amazon 一致
5. `tiktok_product_video`（同 productId）→ 抽样看带货内容：① 播放量是否集中在某条品牌自投视频（=buzz是买的不是自来水，别当真实需求信号）② 视频标签/话题里有没有 Amazon 关键词库没覆盖的用例框架（可能是营销素材新角度，**不直接改产品规格**）

**产出用途（写清楚，别和主判断混）**：
- 与 Amazon 判断一致 → 在 zach payload `ch01.decision_funnel` 加一步 `stage:"gate"`，标注"跨平台交叉验证：TikTok独立确认同一时序判断"，降低"看错了"的不确定性，**不改变 Go/No-Go 分数**
- 与 Amazon 判断不一致（TikTok 在涨而 Amazon 在跌，或反之）→ 这是需要人工判断的分歧点，值得展开查为什么，不要自动采信哪一边
- 内容用例发现 → 存档进项目文件夹，标注给营销/主图阶段用，不进 RFQ 规格
- **不要**把 TikTok Shop 销量/视频热度直接当"需求验证"或纳入 TAM 测算——尤其一条视频远高于其余时，大概率是付费投流不是自然热度，会高估真实需求

### A5b Reddit VOC 探针（可选，A5 的姊妹项，先探针再决定要不要建抓取盘）

> 实测(magic arm案例，2026-07，三轮)：①泛内容创作者社区+类目行话关键词 ②垂直摄影器材圈子(同关键词) ③TikTok调研挖出的"真实受众"圈子(吃鸡/摩托车/路亚钓鱼)+换成品牌名/痛点自然语言关键词——13个subreddit、890篇帖子+156条评论(1046行)、两套关键词策略，"camera clamp/mount"相关话题命中密度稳定收敛在**0.7-1.3%**，三轮独立验证换子版、换关键词策略都没能突破天花板。**结论：Reddit VOC 只对讨论密度本身已经够高的品类有效**（如 cell_phone_tripod/vlogging_kit 抓取盘已验证）。专业器材零件级行话（如"magic arm"）大众创作者根本不这么说话（更常说"my rig"/品牌名，或直接甩产品链接不写长文），密度天生低，换子版/换关键词救不了。

**教训1——先探针再建盘**：不要一上来就搭完整抓取脚本+跑多个 subreddit。先花 1 次小范围搜索判断这个候选的话题是否"大众化到会被人自然吐槽"，密度明显低（<2%）就不值得建抓取盘，改用 A5(TikTok) 或直接跳过这一步。

**教训2——跨渠道受众画像不能直接迁移**：本案例曾假设"TikTok内容里活跃的圈子(吃鸡/摩托/钓鱼)在Reddit上也会更活跃"，实测反而不如原本的专业摄影垂直社区(gopro/videography/cinematography)——TikTok内容消费者和Reddit深度吐槽者不是同一批人。换渠道要基于该渠道自己的证据独立验证，不能假定另一渠道的受众画像可以照搬。

**用途**：与 A5 一样是风险审计，不是发现工具；结果一致时同样只在 `ch01.decision_funnel` 加一步 `gate` 降不确定性，不改分数。挖到的孤立弱相关信号（如与目标形态不同细分场景的求助帖）可记为远期探索方向，不改当前候选规格。

### A5c Google Trends 交叉验证（可选，低成本，用 pytrends 库直查）

> 实测(magic arm案例，2026-07)：`pytrends`（Python库，已装，直查 Google 内部接口，不需要浏览器渲染）查"magic arm"近5年US趋势，发现真实分歧——Google Trends 显示2026年3月下旬起暴涨、4月中旬见顶(5年最高)、之后回落到基线；同期 Amazon 站内 keyword_trend 全程走平无异常。**两个数据源没有互相印证，出现分歧。**

**执行方法（约2-5分钟/候选，免费无API key）**：
```python
from pytrends.request import TrendReq
pytrends = TrendReq(hl='en-US', tz=360)
pytrends.build_payload([核心词], cat=0, timeframe='today 5-y', geo='US')
df = pytrends.interest_over_time()          # 主趋势
related = pytrends.related_queries()        # 排歧义 + 找rising噪音
```

**两个必查步骤，跳过会踩坑**：
1. **`related_queries()`先查一遍**，确认相关查询词确实是目标品类语境（排除同名词歧义，如"magic arm"如果查出来的相关词是游戏角色/其他领域，说明词选错了/查询词有歧义，不能用）。
2. **rising 榜的高百分比默认当噪音，核对绝对量级再信**——实证：magic arm案例中"ifootage magic arm" rising +29750%，看着像新竞争者暴起，但把它和主词放进同一个 `build_payload` 对比后，其绝对量级在0-100尺度上全程显示0，是"从极小基数放大"的假象，且核对 Amazon Top100 该品牌完全没上榜，判定为噪音非真实信号。

**分歧判读（比数字本身更重要）**：Google Trends 统计**全网泛搜索**（含资讯类"这是什么/怎么用"的好奇心搜索，媒体报道/展会/测评视频都能推高），Amazon `keyword_trend` 统计**站内购买意图过滤后的搜索**——两者出现分歧不代表谁错了，是**统计的人群不同**。**分歧时默认更信 Amazon 站内数据**（因为它直接对应我方能承接的购买流量），Google Trends 只作"是否有更大范围行业热度"的参考背景，不覆盖 Amazon 判断，也不改 Go/No-Go 分数。

**用途**：同 A5/A5b，风险审计非发现工具；分歧但已判读清楚时，在 `ch01.decision_funnel` 加一步 `gate` 说明"分歧已核实为统计人群不同，Amazon数据为准"，不改分数。

## B · 交给 zach 深挖必须覆盖的维度（Go/No-Go 打分输入）

这些是完整量化版，blue-ocean 快筛过关后，写进 zach payload 要求覆盖；zach 评分卡应吸收为打分维度。

| 维度 | zach 要产出什么 | 影响哪个评分维度 |
|---|---|---|
| **IP 完整分析** | 对 A1 的雷做 claim chart（逐条比对权利要求）+ 规避设计确认 + RFQ 要 FTO 承诺函 | 入场壁垒 |
| **广告经济/启动现金** | CVR(2/4/6%)×CPC 矩阵、盈亏平衡CVR、打平所需自然占比、到50评广告投入、爬坡月数 | 盈利能力（overhead 此项 8.0→6.5） |
| **形态评分天花板 + 形态分叉** | 同用例各形态 rating 分布；给出形态选择（如带灯 vs 纯机械 A/B 打样建议） | 竞争 / 产品定义 |
| **痛点双口径** | 目标形态本体评论优先；拿不到时标注语料来源（父体/别形态），必要时双口径并陈，说明形态差异 | 产品定义 / 需求清晰度 |
| **TAM 形态封顶** | 三角验证（搜索漏斗 / 节点天花板 / 标杆实证），单SKU天花板 + 全市场 + 年毛利量级 | 市场规模 |
| **季节性 → RFQ 交期** | 峰谷月历 + 交期倒推能否赶下一个峰 | 时序可行性（行动项） |
| **产品定义按"评分杀手"排序** | 痛点按"最致命的评分杀手"排优先级，不是罗列（overhead：LED电气可靠性 > 刚性 > 亮度） | 产品定义 |

### 交接话术模板
> 「已过可行性快筛（IP:__ / 稳态:__ / 形态天花板:__ / 现金量级:__）。深挖请务必覆盖：① IP claim chart ② 广告现金/CVR生死线模型 ③ 同形态评分天花板+形态分叉 ④ 痛点语料代表性(本体评论/双口径)，按"评分杀手"排优先级不是罗列 ⑤ TAM形态封顶 ⑥ 季节交期。评分卡的盈利能力/入场壁垒维度要吸收①②③。」

## 已知数据陷阱（Sorftime）
- product_reviews 常返回**父体/别变体**评论，非目标 ASIN 本体 → 痛点会偏，务必看"评论产品的属性"字段，标注来源。
- product_search 是**相关性+销量排序**，非纯节点过滤，会混入噪音 → 形态天花板用 property_name 过滤 或 category_report(node_id) 精确化。
- product_trend 仅**月度**颗粒，逐日拆促销要靠快照/后台。
- 目标 ASIN 本体评论可能因积分/父体聚合拿不到 → fallback：用同形态最接近的高评论竞品 + 明确标注非本体。
