# 缝隙签名（gap signature）评分卡

一个真缝隙 = **需求在涨（段1）+ 竞争尚浅（段2）**，加分项让它更值得做。

## 段1 · 需求侧（keyword_growth_scanner.py 自动算）

| 维度 | 权重 | 怎么读 |
|---|---|---|
| YoY 增速 | 50% | +80%→10分 / +30%→8 / 0→4 / 跌→低。**最重要**——缝隙先看需求涨不涨 |
| 月搜量 | 30% | 可做带 3k-80k 最高；<3k niche太小；>150k 多半红海/大词 |
| CPC | 20% | <$1→10 / $1-1.5→8 / >$2→4。低=投放便宜=竞争没那么贵 |

段1分 ≥7 且 🟢涨 → 进段2。

## 段2 · 供给侧（人工核 keyword_detail + product_search）

| 信号 | 浅竞争长这样 |
|---|---|
| 首页评论数分布 | <100评 占比高（如 overhead 31.6%）= 新品能上首页 |
| 头部评论墙 | 头部仅几百-千余评（非数万）= 护城河浅 |
| 有无统治者 | 无单一品牌横跨该用例 = 分散 = 可切入 |
| 价格带 | 非 $10 商品战；能撑 50%+ 毛利 |
| **形态天花板 vs 搜索池** | 我方能承接的**形态**在节点里能做多少量？节点头部若是别的形态（如 overhead 节点被 $16 廉价夹灯占），我方形态天花板可能远低于搜索池/节点总量。TAM 按形态封顶，不按搜索池 |
| **同形态评分天花板** | 同一用例不同形态，评分天花板可能差 0.2-0.4 星（overhead：无灯臂 4.6-4.8，带灯 rig 封顶 ~4.5）。评分低的形态=工程难做对=护城河机会，但也=评分负债。选形态时一并看 |

## 加分项（不是必须，但加分）

- **痛点可工程解决**：抽头部差评，看不满是否来自"没人做对"（可质量位移）而非"品类满足不了"
- **能力圈贴近**：和现有供应链/listing 能力相关 → 执行风险低
- **无电子件/合规简单**：制造门槛低

## 排除项（一票否决，交 zach 前必查 → 见 feasibility_gate.md）

- 需求在跌（YoY 明显负）/ 是促销尖峰堆的假增长（拆促销窗口后看稳态）
- 已有数万评的深护城河统治者
- gated 类目 / 危险品 / 超尺寸吃运费
- **专利雷区（升级为可执行快筛）**：核心形态/机构在 USPTO/Google Patents 有生效的**实用新型**阻挡专利，且权利人是积极维权方（如 overhead 磁吸踩 Videndum US11796126B2）→ 换方案或换形态规避。快筛做法见 feasibility_gate.md
- **启动现金撑不住**：评论 0→50 阶段广告烧钱 × CVR 若使标的从"试水"变"重投入"，超出账户弹药能力 → 降级或缓做

## 判定

- 段1✅ + 段2✅ + 可行性闸门✅ → 候选，进 shortlist → 决策点① → zach 深挖
- 段1✅ + 段2✅ + 闸门❌（IP雷/现金撑不住）→ 缝隙真但做不了，弃或换形态
- 段1✅ + 段2❌（竞争已厚）→ 红海，弃
- 段1❌（需求跌/平/促销假涨）→ 默认弃，别浪费段2。**但若段2/VOC信号显示"在位者集体摆烂"（连头部品牌评论都密集复现同一组工程失败痛点、形态IP干净、稳态销量真实非促销堆），存在路径B——见下方「路径B：红海质量位移」**

### 路径B：红海质量位移（段1不过时的可选分支，非自动触发）

> 实证：magic arm/桌夹臂案例（2026-06/07）。19词段1扫描 0个通过（🟢涨且≥7分=0/19），按常规判定该弃。但P2快筛发现头部品牌（含类目leader SMALLRIG）差评全中同一组工程失败（撒手就垂/夹口装不下大手机/QC滑丝），稳态销量真实存在（非促销堆）——用户拍板转向"靠制造/QC抢在位者摆烂的份额"打法，照样走完P2-P4→zach深挖→CONDITIONAL GO 6.2→RFQ。

**触发条件**（缺一不可，都要有证据支撑）：
1. 段1❌，但不是断崖式归零（仍有真实稳态销量，非虚无品类）
2. 段2/头部竞品差评显示**集体性、跨品牌复现的工程缺陷**（不是个别烂品，是连leader都没做对）
3. 形态IP干净 / 制造门槛可控（质量位移的护城河来自工程执行，不是专利或红利，做不出真差异化就白搭）

**这是决策点，不是自动判定**：段1❌出现路径B迹象时，**把两条路径都摆给用户选**（①弃换方向 ②质量位移深挖 ③再扩词确认段1真弃），不要替用户拍板。选了路径B之后，P2-P4/A5系列/交zach的流程和"正常蓝海候选"完全一样，只是 `ch01.decision_funnel` 里要如实标注"这不是上升缝隙，是红海位移"（见 handoff.md 映射表），不能包装成蓝海故事。
