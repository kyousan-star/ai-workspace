# Amazon Image Planner v3 · 单次图片规划执行层

每张图输出 **A/B 双 prompt**：
- **方案 A**：传统功能型（讲清楚卖点）
- **方案 B**：停顿感型（让用户先停下来），三模板：T09 产品存在感 / T10 反差感 / T11 焦虑场景叙事

**v3 相对 v2 的变化**：轻量化（产品资料不齐也能启动，用户口述即可）；新增 Phase 1.5 图序排序（独立模式下由用户的卖点优先级驱动图序，v3 不替用户定优先级）；新增 workflow 模式调用契约。定位判断详见 `WHEN_TO_USE.md`。

---

## 运行模式判定（最先做）

| 信号 | 模式 |
|------|------|
| 上游传入"图序（已锁定）+ 卖点优先级 + 手段候选库 + 否决手段/视觉禁区"（amazon-image-workflow 阶段4） | **workflow 模式**：跳过 Phase 1 / 1.5 / 2，直接 Phase 0 → Phase 3 |
| 用户直接触发，无上游上下文 | **独立模式**：Phase 0 → 5 全流程 |

**workflow 模式附加要求**：
- 图序和卖点优先级是 Gate 1/2 用户确认锁定的，**不得自行调整**
- 否决手段、视觉禁区必须写入每个 prompt 的 STRICTLY AVOID
- 每张图的构图手段标注来源：「引用竞品手段库 Part 2 第 X 条」或「原创构图」

---

## Phase 0：停顿感评估（内部完成，不提问）

判断产品类型，确定每张图的方案 B 推荐模板：

| 产品特征 | 推荐方案 B 模板 |
|----------|---------------|
| 小型手持品、美妆、女性生活方式 | T09 产品存在感型 |
| 有强烈品类刻板印象（电竞/工具/收纳） | T10 反差感型 |
| 解决具体痛点/风险（防护/防丢/防漏） | T11 焦虑场景叙事型 |
| 纯技术参数驱动、理性决策品类 | 方案 B 仍用功能型模板，不强行套钩子 |

这是推荐，独立模式下 Phase 2 等用户确认后再执行；workflow 模式下直接采用（上游已锁定卖点，模板匹配属执行细节）。

---

## Phase 1：轻量 Discovery（独立模式）

**有 workspace 文件时**扫描：产品信息 PDF、VOC 报告、实拍图、竞品参考（同 v2 的文件模式）。
**没有文件时不阻塞**：直接用用户口述的产品名称、核心功能、目标用户；缺的信息在 Phase 2 一次问齐。竞品手段仅做轻量提取（有竞品图就归纳构图手段候选，没有就跳过），**不做完整竞品研究**——需要完整研究的场景应走 amazon-image-workflow。

---

## Phase 1.5：图序排序（独立模式，v3 新增）

输入是用户给的**卖点优先级**（最关键输入，v3 不替用户决定"最重要讲什么"）：

1. 按优先级把卖点映射到图片槽位：副图1 承接第一优先级（搜索结果页最吃第一眼），依次往后
2. 场景图/人群图/参数图的穿插按"认知节奏"排：钩子 → 场景带入 → 硬参数 → 信任（包装/售后）
3. 输出图序表（槽位 / 卖点 / 图型 / 方案B模板推荐）给用户确认——确认后才进 Phase 3

用户没给优先级时先问，不要替用户排。

---

## Phase 2：Clarification（独立模式，最多 4 个问题，1 次批量提问）

1. 图片组合范围（几张副图 + 几张 A+ 图）？
2. 视觉风格：高端科技感 / 暖色生活方式 / 极简 / 大胆用色？
3. 品牌色或排版约束？
4. 每张图的方案 B 模板推荐确认（Phase 0 的结果 + 理由，允许逐张调整）

---

## Phase 3：应用规则与模板

**方案 A**：功能型构图逻辑（降噪/即插即用/长距离/续航/多对一/工艺细节等）
**方案 B**：读取 `./references/composition-templates-v2.md`，按确认的模板写 prompt；T10 反差感必须先完成"刻板印象→反向元素"推导才能写 prompt

**所有 prompt 必须执行**（读取 `./references/anti-pitfall-rules-key.md`）：
- 产品形态描述在 planning 开始时锁定，A/B 两套方案全程一致
- 金属零件用银/铬色，不用纯金/铜
- 不画手指按键，改用按键发光+箭头
- 远景人物占画面 1/10-1/8
- 每个 prompt 末尾加 STRICTLY AVOID 列表（8-15 项；workflow 模式必须并入上游否决手段+视觉禁区）
- 指定拍摄设备（`shot on Sony A7 with [X]mm lens`）

---

## Phase 4：输出 HTML 规划文档

单一 HTML 文件，每张图 A/B 双 Tab（结构、全局卡片、技术规范与 v2 相同）：

- 每张图卡片：讲述逻辑 / 构图思路 A|B / 图内文字清单 A|B / 参考图 / 验收标准 A|B（B 额外检查：反差合理？产品仍是主角？钩子文案≤6词？）/ Prompt 双 Tab 各带 Copy 按钮
- 顶部全局卡片：统一产品形态约束、统一色板、统一排版规范、A/B Test 使用指引（A 主站稳定转化；B 放最吃第一眼的槽位，test 4-6 周对比 CTR/转化率；B 的 CTR 升转化降 = 钩子有效、产品描述需跟进）
- workflow 模式额外：每张图卡片标注手段来源（Part 2 第 X 条 / 原创）
- Sticky 导航、原生 CSS+JS 无外部依赖、Tab A 白底 / Tab B 浅暖黄 `#FFFBF0`、无 emoji

**文件命名**：`{产品名}_生图方案v3_{YYYY-MM-DD}.html`，保存到用户 workspace 文件夹（workflow 模式按 workflow 的项目文件树落位）。

---

## Phase 5：Hand-off

1. 文件路径（computer:// 链接）
2. 生图顺序建议：方案 B 先跑高风险图（T10/T11 AI 理解难度大）
3. 出图后调用 `product-consistency-checker` 检查 A/B 两套产品形态一致性（该 checker 在 amazon-image-toolkit 内）
4. A/B test 槽位建议
5. workflow 模式：交还控制权给 workflow 阶段5（生成评估），不自行推进

---

## 版本关系

- **v3 = 当前主版本**（本 skill）；v2 仍安装可用（触发词"用v2规划图"），差异仅在 v3 的轻量启动、Phase 1.5 图序排序和 workflow 契约——模板库和防坑规则两版共用同一套内容
- 修改 T09-T11 模板或防坑规则时，v2/v3 的 references 需同步（同源文件，暂为复制关系）
