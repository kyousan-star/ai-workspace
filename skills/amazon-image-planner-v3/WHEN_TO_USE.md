# 什么时候用这个工具

## 一句话定位

**Amazon Image Planner v3 是单次图片规划的轻量执行工具。**

你告诉它产品信息和卖点优先级，它输出每张图的 A/B 双 prompt——方案 A 功能型，方案 B 停顿感型，可以直接发给 AI 生图工具使用。

---

## 适用场景

### ✅ 用这个工具，当你面对的是：

**已知图序，需要快速出 prompt**
- 你已经想好每张图要讲什么（哪张讲降噪、哪张讲续航、哪张讲场景）
- 只需要把这个图序转成可执行的生图 prompt
- 不需要跑完整的竞品研究和策略流程

**单张或少量图片的规划**
- 只需要规划 1-3 张图，不是整套图片矩阵
- 比如只做一张场景合成图，或者改一张副图的表达方式

**轻量启动，不想跑完整 workflow**
- 时间紧，需要快速出方案
- 产品资料不齐全，但想先把 prompt 方向定出来
- 测试某个卖点的视觉表达效果

**A/B 双方案测试**
- 想给同一张图出两个方向的 prompt，一个稳健功能型，一个有停顿感的钩子版
- 方案 B 的停顿感模板：T09 产品存在感、T10 反差感、T11 焦虑场景叙事

**workflow 阶段4的执行层**
- 如果你在跑 amazon-image-workflow，阶段4会直接调用 v3
- 这时不需要手动启动，workflow 会传入已锁定的图序和上下文

---

## 不适用场景

### ❌ 不要用这个工具，当你：

- 还不知道图序是什么、不知道每张图要讲什么（先用 `amazon-image-workflow` 把策略和图序定清楚）
- 需要从竞品研究开始、做完整的新品视觉方案（用 `amazon-image-workflow`）
- 需要跨多天推进、多人协作、需要 Gate 确认机制（用 `amazon-image-workflow`）
- 需要诊断现有在售品图片的问题（用 `amazon-image-optimizer`，开发中）

---

## 运行环境

**claude.ai 对话**（单 session，无需文件系统）

直接在 claude.ai 对话框里触发，告诉它产品信息和卖点优先级就可以开始。

触发词：「用v3规划图」「图片规划」「规划副图」「A/B生图方案」「做双方案图规划」

---

## 使用前你需要准备

| 必须有 | 有更好 |
|---|---|
| 产品名称和核心功能描述 | VOC 痛点报告 |
| 你定义的卖点优先级顺序 | 竞品图片文件夹 |
| 需要规划几张图 | 实拍白底图 |
| 视觉风格偏好 | 品牌色和排版约束 |

**最关键的输入是卖点优先级**——你要先想清楚"这套图最重要的是讲什么、其次是什么"，v3 不替你做这个决定。

---

## 与 amazon-image-workflow 的关系

```
amazon-image-workflow（完整项目）
    研究 → 策略 → Gate1 → 图序 → Gate2
                                    ↓
                        amazon-image-planner-v3（执行层）
                        输出 A/B 双 prompt
```

v3 是 workflow 的执行层，也可以完全独立运行。

**两者的核心区别：**

| | amazon-image-workflow | amazon-image-planner-v3 |
|---|---|---|
| 适合什么 | 新品 0→1 全链路 | 已知图序，快速出 prompt |
| 运行环境 | Claude Code / Codex | claude.ai 对话 |
| 需要多久 | 多个 session，有 Gate 确认 | 单次对话，30分钟内 |
| 竞品分析 | 完整三部分结构化分析 | 轻量扫描，提取手段候选库 |
| 图序来源 | Gate 1 用户确认锁定 | Phase 1.5 你来排序 |
| 输出 | 完整项目文件树（10个阶段） | 单个 HTML 规划文档 |

## 快速判断

```
我知道每张图要讲什么，需要出 prompt
    → 用 v3，直接在 claude.ai 触发

我还不知道图序，需要从研究开始
    → 用 amazon-image-workflow

我只需要改某张图的 prompt 方向
    → 用 v3，告诉它这张图的任务

我在跑 workflow，已经过了 Gate 2
    → workflow 会自动调用 v3，不用手动切换
```

---

## 关于 A/B 双方案

每张图会出两个 prompt：

**方案 A（功能型）**：讲清楚卖点，用户一眼看懂这张图在说什么。适合主站稳定转化。

**方案 B（停顿感型）**：让用户先停下来，再传递卖点。适合放搜索结果页最吃第一眼的槽位做 A/B test。

方案 B 有三种模板，v3 会根据产品类型推荐，你来确认：
- **T09 产品存在感**：小型手持品、美妆、女性生活方式
- **T10 反差感**：有强烈品类刻板印象的产品
- **T11 焦虑场景叙事**：解决具体痛点/风险的产品
