# Report Design

## 目标

把已有分析内容转成结构清晰、视觉可读、便于汇报和复盘的报告交付物。核心原则：

- 不牺牲内容完整度。
- 首屏先给结论和关键指标。
- 正文保留完整证据、章节、表格、列表和风险口径。
- 视觉增强服务于阅读和决策，不做空洞装饰。

## 默认输出策略

用户没有特别指定时，优先生成 **完整可视化 HTML**：

```bash
python3 /Users/lihuan/.codex/skills/report-design/scripts/render_full_markdown_report.py source.md output.html
```

完整 HTML 应包含：

- 首屏决策/摘要看板
- 左侧或顶部目录
- 完整正文
- 完整表格
- 状态高亮
- 风险与口径说明
- 移动端和桌面端都可阅读的布局

## 何时用简版 Dashboard

只有用户明确说“只要 dashboard / 看板 / 一页摘要 / 老板快速看”时，才使用简版：

```bash
python3 /Users/lihuan/.codex/skills/report-design/scripts/render_dashboard.py input.json output.html
```

## 支持格式

### HTML

首选格式。适合：

- Markdown 报告转网页
- 数据分析报告
- 市场调研报告
- 周报/月报
- 项目复盘
- 投资研究
- 品类/竞品/运营报告

### PPTX

适合会议汇报和老板 review。优先使用 Presentations Skill 或 python-pptx。结构建议：

1. 封面
2. 一页结论
3. KPI/关键指标
4. 核心发现
5. 证据和数据页
6. 风险与口径
7. 行动计划

### PDF

适合归档、发送、正式报告。优先从 HTML 打印/导出 PDF，或用 PDF Skill 生成并渲染检查。

### DOCX

适合偏文档型交付，如调研报告、咨询报告、正式说明书。优先使用 Documents Skill。

### XLSX

适合数据表、模型、预算、评分卡、行动追踪表。优先使用 Spreadsheets Skill。

### Markdown

适合保留原始可编辑版本。若用户要求“重新排版 Markdown”，保持标题层级、表格和列表完整。

## 报告结构规则

完整报告按以下层次组织：

1. **首屏摘要**：一句话结论、状态、核心指标、关键判断。
2. **目录导航**：让读者能跳转到章节。
3. **完整正文**：保留原始章节、表格、列表、引用、脚注。
4. **视觉增强**：表格、评分、Gate、风险、行动项高亮。
5. **自检**：确认没有丢失关键章节或表格。

## 视觉规则

- 面向业务、研究和决策场景，使用克制、清晰、信息密度高的报告风格。
- 不使用装饰性渐变、大片空白、营销海报式 hero。
- 表格必须可读，列多时使用横向滚动。
- 状态类字段使用红黄绿，但不要让页面变得花哨。
- 任何摘要都不能替代完整正文，除非用户明确要求摘要版。

## 交付前自检

- 原始报告的一级/二级/三级标题是否完整保留。
- 表格数量是否明显少于原文；如少了，必须解释原因或修复。
- 是否保留关键数字、风险、行动建议。
- HTML 是否能被解析。
- 输出文件是否放在用户要求的位置。

## Amazon 场景附录（自 amazon-report-design 合并，2026-07-02）

处理亚马逊业务报告（广告/VOC/Listing/竞品流量/品类/选品/周报/复盘）时，在通用规则之上追加：

**配套关系**：本 Skill 不替代分析 Skill——先由 amazon-ad-optimizer / amazon-review-voc-analysis-v3 / amazon-listing-v2 / competitor-traffic-battle / amazon-category-analysis / zach-product-research 完成计算和诊断，再用本 Skill 做报告压缩、版式组织、图表表达和行动清单。

**报告结构（正式报告按此顺序）**：
1. **结论先行**：一句话判断当前状态（如"广告亏损主要由 12 个高花费无转化词造成，先止损再扩量"）
2. **状态灯**：绿/黄/红，最多 5 个关键指标
3. **核心 KPI**：ACOS、ROAS、Spend、Sales、Orders、CTR、CVR、CPC、TACOS、评论样本量、痛点占比、关键词覆盖、竞品流量份额等（按报告类型取用）
4. **问题排序**：按利润影响、确定性、执行难度排序，不按发现顺序
5. **机会清单**：只保留能执行的机会，避免泛泛建议
6. **行动计划**：今天做什么、7 天内做什么、30 天验证什么
7. **风险和数据局限**：样本不足、抓取覆盖、时间范围、口径不一致必须明示

**视觉补充**：面向运营和老板汇报用密集但清晰的业务仪表盘风格；指标卡最多 6 个，每卡必含数值、方向/状态、业务解释。
