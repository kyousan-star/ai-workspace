# V1.3 Production Safety Status

Date: 2026-07-20

Status: `CORE IMPLEMENTED / LIVE DB AND BROWSER PASS`

## Decision

The workbench no longer treats repeated ImageGen calls as the default way to
produce exact Amazon product images. ImageGen is a concept/background provider.
Reviewed real product pixels are the source of truth for final Listing and A+
assets.

## Implemented

- `production-route.1` Gate for P1, P2, direct jobs, CLI, HTTP, and MCP.
- Four routes: `deterministic`, `composite`, `concept_only`, and `blocked`.
- Missing required view, non-transparent source, unapproved source quality, or
  missing composite background returns blockers and capture requests.
- Composite backgrounds require an explicit product-free review flag; a file
  path alone is not sufficient.
- Exact-product routes require `manual_import` and cap attempts at one.
- Pillow compositor preserves aspect ratio, never generates product pixels, and
  writes source hashes and placement bounds in `production-provenance.1`.
- Product identity, geometry, and proportion failures hard-stop the slot and
  cancel queued, leased, or awaiting-import jobs.
- Studio UI displays the production route and exposes deterministic execution.
- MCP exposes 40 tools, including route preflight, deterministic execution, and
  production failure recording.
